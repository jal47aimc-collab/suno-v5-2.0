# Scripture Music Master Document (V5-Ready Placeholder Edition)

---

## Introduction

This document defines the modular framework for scripture-based music generation using placeholder logic. It is designed for integration with Suno AI and future metadata libraries, including 90s underground rap sonic vocabularies. All examples use placeholders to support flexible reuse and dynamic metadata injection. No references to vinyl crackle or tape hiss are included.

---

## Metadata Placement Standards

### Tempo Declaration Logic

- If the song maintains a single consistent tempo, declare `bpm:` in the top-level `[studio: ...]` block.
- If the song includes tempo changes, omit `bpm:` from the top-level block and declare `tempo:` inside each song section.

### Key, Groove, Theory, and FX

- These may be declared globally in the `[studio: ...]` block if consistent.
- Otherwise, define them per section using placeholders.

---

## Section Labeling Rules

- All song sections must begin with a lowercase bracketed tag.
- Examples: `[v1: ...]`, `[hook1: ...]`, `[narration1: ...]`
- Do not use `Verse 1`, `Hook`, or other non-bracketed labels.

### Master Metadata Document — Song Metadata Standard

### Purpose
Define a single authoritative metadata standard for scripture-to-song outputs. Enforce one-bracket-per-section explicit metadata, a normalized vocabulary, and production rules so LLMs and downstream generators produce consistent, repeatable results.

---

### Studio Registry and Genre Tokens

#### Studio Registry

#### Genre Tokens

- **militant boom-bap** — aggressive boom-bap production with militant rhythmic emphasis

---

### Top-level song header (one bracket pair only)
[studio: <studio name or neutral preset>; genre: <genre>; persona info: <none|persona>; default meter: <4/4>; stylebox: <stylebox tokens>; no_tape_hiss|allow_vinyl_crackle; production_notes: <short free-text only>; key: <key>; bpm: <BPM>; verses: <range optional>]

Rules
- Appears once at the top of each song file.
- Free-text limited to production_notes only.
- Studio must use a registered studio slug when available; use studio registry for canonical names.

---

### Section metadata (single-bracket explicit metadata rule)
All metadata for a section must be inside one bracket pair at the start of the section. The first field MUST be the section_label.

[section_label: <intro|narration1|v1|v2|hook1|bridge|outro>; voice: <raw_flow|spoken|theatric_deep|sing_vocal|choral|elder_speaker|gritty_testimony_voice|smooth_sermon_voice|urgent_prophet_voice|elder_wisdom_voice|echoed_prayer_voice>; bars: min–max; tempo: <BPM>; key: <Key>; theory: <controlled_theory_token>; groove: <descriptor>; cadence: 1|3; insert_anchor: <kick|snare|kick_on_1_snare_on_3|sub_swell|low_bass_hum>; loop_friendly: yes|no; sonic: <primary;secondary;fx>; fx: <short list>; persona info: <none|persona_token>; stylebox: <stylebox tokens>]

Required fields
- **section_label** (first)  
- **voice** (explicit)  
- **bars**  
- **tempo** or inherit from song top if not provided  
- **sonic** (must include primary;secondary;fx)

Prohibitions
- No extra bracket pairs for the same section.
- No informal placeholders like hint, token, ish.
- No production free-text inside section metadata; production_notes only at song top.

Examples
- Correct:  
  [v2: voice: gritty_testimony_voice; bars: 10–16; tempo: 78; key: A minor; theory: modal_minor_gospel; groove: boom_bap_slap; cadence: 3; insert_anchor: kick_on_1_snare_on_3; loop_friendly: yes; sonic: sub_bass_foundation;warm_rhodes;cinematic_echo_trail; fx: tape_delay_warm; persona info: jal_the_messenger; stylebox: early90s_gritty;no_tape_hiss]
- Incorrect:  
  [V2: verses 6–13; tempo: 78; theory: dorian_hint; groove: half_time_groove]

---

### Narration rules
- Use narration sections where the KJV text supplies scene-setting or stage direction.
- Section label format: narration1, narration2, etc. Narration always precedes the musical section that follows.
- Default voice for narration: elder_wisdom_voice or theatric_deep depending on file; include insert_anchor such as low_bass_hum and a short sonic cue.
- Keep narration short (1–4 lines) unless explicitly requested as extended spoken-song.

Example
[narration1: voice: elder_wisdom_voice; bars: 2–4; tempo: 86; key: A minor; theory: none; groove: spoken_pulse; cadence: 1; insert_anchor: low_bass_hum; loop_friendly: yes; sonic: ambient_pad_layer;low_bass_hum;fx: room_plate; persona info: none; stylebox: spiritual_lofi;no_tape_hiss]

### Song-level rules and verse-range tracking
- Verse ranges are tracked only at the song-level header: use verses: 1–17 in top bracket when desired.
- Favor studio-level declarations for key, groove, theory, fx, and bpm when consistent; declare per-section when they change.
- Sections reference bars and must be split so bar counts match the studio arrangement.

---

### Logic Rules and Best Practices
- **Studio-level tempo** Declare a single steady tempo using bpm in the top-level studio header when tempo is consistent across the entire song.
- **Per-section tempo** If tempo varies, remove bpm from the studio header and declare tempo inside each section.
- **Global metadata** Declare key, groove, theory, and fx in the studio header when consistent; declare them per-section when they change.
- **Section tags** All sections must start with a lowercase bracketed tag: intro, narration1, v1, hook1, outro, etc.
- **Literal scripture default** Sections should contain verbatim KJV scripture for the selected chapter unless the producer explicitly requests non-scripture inserts. Non-scripture must be flagged and stored as separate template variants.
- **Narration usage** Use narrationX for spoken, expository, or free-tempo passages; use vX for scripture lines that map to rhythmic phrasing.
- **Character economy** Favor studio-level declarations to reduce per-section header size and preserve space for tokens like scent_pad.
- **Stylebox application** Apply a stylebox preset at the studio or section level by listing its tokens inline; stylebox tokens are semicolon-delimited.
- **Token stacking** Stack sonic and FX tokens separated by semicolons inside the sonic: field; avoid invented words inside token lists unless explicitly approved.
- **Persona consistency** Assign a persona token for the overall song and override if a section calls for a different narrative POV.
- **Exclusions** By default, **no_tape_hiss** is enforced in styleboxes unless allow_vinyl_crackle is explicitly present.
